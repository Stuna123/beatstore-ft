# 🔥 EXPLICATION DOSSIER PAR DOSSIER

---

## 📁 `server.js` — ENTRY POINT

Responsabilités :

* Charger `.env`
* Connecter MongoDB
* Configurer Express
* Monter les routes
* Lancer le serveur

👉 **Aucun business logic ici**

---

## 📁 `config/`

### `db.js`

```js
Connexion MongoDB (mongoose)
```

### `stripe.js`

```js
import Stripe from "stripe";
export default new Stripe(process.env.STRIPE_SECRET_KEY);
```

👉 **Config isolée = clean + testable**

---

## 📁 `models/` — SCHEMAS MONGOOSE

### `User.js`

* email
* password (hashé)
* role (`user` | `admin`)

### `Product.js`

* name
* price
* image
* brand
* stock

### `Cart.js`

```js
user
items [
  product (ref)
  quantity
]
```

### `Order.js`

```js
user
items [
  product
  quantity
  price (figé)
]
totalPrice
status (pending | paid)
```

🔥 **Prix figé = règle e-commerce PRO**

---

## 📁 `controllers/` — LOGIQUE MÉTIER

### `authController.js`

* register
* login

### `productController.js`

* createProduct (admin)
* getProducts
* getProductById

### `cartController.js`

* getCart
* addToCart
* removeFromCart

### `orderController.js`

* createOrder
* getMyOrders
* getAllOrders (admin)

### `paymentController.js`

* createCheckoutSession
* stripeWebhook

👉 **Un controller = une responsabilité**

---

## 📁 `routes/` — ROUTING EXPRESS

Chaque route :

* appelle un controller
* applique les middlewares

Ex :

```js
router.post("/create-session", authMiddleware, createCheckoutSession);
```

🔥 **Lisible, scalable**

---

## 📁 `middleware/`

### `authMiddleware.js`

* Vérifie JWT
* Injecte `req.user`

### `adminMiddleware.js`

* Vérifie rôle admin

👉 **Sécurité centralisée**

---

## 📁 `utils/`

### `generateToken.js`

* Génère JWT
* Utilisé par authController

---
